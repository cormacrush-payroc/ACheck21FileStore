Regular expression patterns consist of meta-characters, which are characters that represent other characters. These meta-characters tell the regular expressions parser how to scan the input string and what to search for. Regular expressions are used by search engines, search and replace dialogs of word processors and text editors, in text processing utilities, and in lexical analysis. Many programming languages provide regex capabilities either built-in or via libraries, as it has uses in many situations. X9Assist supports Java RegEx as an additional search option from various tools such as Search/Replace.

You can find many online tutorials that can be helpful to learn more about Java RegEx capabilities. There are also online tools that can be used for interactive development and testing of RegEx strings to meet a specific objective. Here is a brief, high level summary of some RegEx functions.

Greedy searches – regex matches are usually greedy so they match as much as possible, this means using a wildcard can match everything.  A* would match all of the A’s in AAAAAAAAAAAAAAAAAAAAAB, A+ would also match them, A would match the first one, A{10} would match the first 10 and so on.

(?<= )  - this looks for a match to whatever terms are after the = but does not return it, when put in front of a search it has to match this first.  Effectively acts as a left boundary.

(?= ) - this looks for a match to whatever terms are after the = but does not return it, when put after of a search it has to match this last.  Effectively acts as a right boundary.

\d = any digit

[A] = match any A

[ABC] = match any A, B, or C character

[0-9] = match any digit from 0-9

[0-9]+ = match all digits in a row, minimum 1

[0-9]* = match any number of digits in a row (including none)

^ = start of a line

$ = end of a line

\ = used as an escape character, e.g. \\  matches the \ character

? = after a character or ( )? Makes that term optional (greedy means it will include it if it there)

( ) = group terms and also creates the bracket contents as a variable (variable is referenced as a number based on the order of the opening ( e.g. first ( ) is 1, and so on, can be inside brackets themselves.

\1 $1 = depends on implementation but can be used to reference the value of the corresponding term in brackets

Field Regex Regex Notes
Amount (?<=B)\d+ Matches the part of a string preceded by B that consists of only numbers - it will get all the numbers and stop when it reaches anything not a number.
On-Us (?<=A)[0-9DC]+(?=C|B|$) Matches the part of a string preceded by A, that contains numbers, C, or D and ends with B, C or the end of the line NOTE: May need to update to also include the trailing C
Transit [0-9D]+(?=A) Matches the part of a string that precedes A and has numbers or D.  NOTE: May need to be made more robust to have a preceding A (but not all checks had this), can also be set to be exactly 9 numbers
EPC (?<=^|B)[0-9](?=A) Matches a single number that is preceded by B or the start of the line, and is followed by A.
Aux On-Us [\dD]+(?=CA) Matches the part of a string that consists of digits and D, and is followed by CA. NOTE: May need to allow for C in the string, and not sure if it will always have the C (and it also needs a B).  Sample I have uses C at each end whereas the standard has B at the end.
