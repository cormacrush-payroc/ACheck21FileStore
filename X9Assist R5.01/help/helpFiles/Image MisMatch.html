Image MisMatch allows you to create items where the front images are mismatched with the associated MICR line data (either ICL type 25 or ICLR type 31). This is a helpful process when you want to purposefully create image mismatch conditions to be used for application testing.

When you invoke Image MisMatch, you must first position within your file at the point where you want the image mismatch to begin. You then enter the number of mismatched items that you want to generate. The number of generated mismatch items is limited only by the number of actual items that are in the currently loaded file. In the above example, the file has 1,000 items, and the resulting message indicates that you can generate up to 999 image mismatched items.

The errant image relationships are created by assigning the front image from item “n” to item “n+1”. If you position on the first item in the file, then that front image will be assigned to items one and two. The front image from item two will be reassigned to item three; the front image from item three will be reassigned to item four; and so on.

You can create mismatched items that cross both bundles and cash letters.

Note that the back side images of these items are not changed; only the front images (with show the MICR code line) are manipulated.

You can generate more than one sequence of mismatched items within the same file. You would do this by simply running MisMatch multiple times.

Image MisMatch will launch a new validation process for the resulting image mismatch file, which will remain in memory after the mismatch process has been run. You must save the results to an external file if you determine that the results are beneficial.
